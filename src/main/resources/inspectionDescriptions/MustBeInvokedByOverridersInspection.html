Reports overriding methods that do not call the corresponding super method when the super method is annotated with @MustBeInvokedByOverriders or @OverridingMethodsMustInvokeSuper.

Methods annotated with these annotations establish a contract: every override must invoke the super implementation to preserve the base class behavior. Make sure the description doesn’t just repeat the inspection title.

Example of a problematic override:


class Child : Parent() {
    override fun setup() {       // WARNING: missing super.setup()
        println("Child setup")
    }
}

Correct usage:


class Child : Parent() {
    override fun setup() {
        super.setup()            // required by @MustBeInvokedByOverriders
        println("Child setup")
    }
}
