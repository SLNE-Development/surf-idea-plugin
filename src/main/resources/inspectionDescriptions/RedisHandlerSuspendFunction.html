Reports Redis handler methods annotated with @OnRedisEvent or @HandleRedisRequest that are declared as suspend functions.

Surf-Redis invokes handler methods via MethodHandles and does not support Kotlin coroutine suspension. Declaring a handler as suspend will cause a runtime error because the framework cannot provide a Continuation. If you need asynchronous work, launch a coroutine inside the handler using your own CoroutineScope.

Example:


// ERROR: handler must not be suspend
@OnRedisEvent
suspend fun onUserCreated(event: UserCreatedEvent) {
    delay(100)
}


// Correct
@OnRedisEvent
fun onUserCreated(event: UserCreatedEvent) {
    scope.launch { delay(100) }
}

The quick-fix removes the suspend modifier from the function.
