Reports Redis handler methods that are declared as suspend functions.

Methods annotated with @OnRedisEvent or @HandleRedisRequest must not be suspend functions. The Redis handler framework invokes these methods synchronously using reflection, and suspend functions are not compatible with this mechanism.

Example of problematic code: 


@OnRedisEvent
suspend fun onEvent(event: MyEvent) {  // suspend not allowed
    // ...
}

Correct code: 


@OnRedisEvent
fun onEvent(event: MyEvent) {
    coroutineScope.launch {
        // async work here
    }
}

The quick-fix removes the suspend modifier. If you need to perform asynchronous operations, launch a coroutine inside the handler using your own CoroutineScope.
