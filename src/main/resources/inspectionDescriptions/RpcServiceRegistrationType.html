Reports registerRpcService/unregisterRpcService/createRpcService calls whose service type argument is not a @RpcService-annotated interface.

The RPC runtime resolves the generated service descriptor from the Service type argument. When the implementation is passed without an explicit type argument, Kotlin infers the implementation class — for which no descriptor exists — and the call fails at runtime. Always specify the annotated interface explicitly. 


serverApi.registerRpcService(UserServiceImpl())              // reported — infers UserServiceImpl
serverApi.registerRpcService<UserService>(UserServiceImpl()) // OK
clientApi.createRpcService<UserService>()                    // OK

The quick fix inserts the @RpcService interface implemented by the argument.
