Reports @HandleRedisRequest handler methods with incorrect parameter signatures.

A @HandleRedisRequest handler must have exactly one parameter of type RequestContext<T>, where T is a subtype of RedisRequest. The RequestContext provides access to the incoming request and the mechanism to send a response.

Example of problematic code: 


@HandleRedisRequest
fun handle() { }  // Missing parameter

@HandleRedisRequest
fun handle(context: RequestContext, extra: String) { }  // Too many parameters

@HandleRedisRequest
fun handle(request: MyRequest) { }  // Wrong parameter type

Correct code: 


@HandleRedisRequest
fun handle(context: RequestContext<MyRequest>) {
    val request = context.request
    context.respond(MyResponse(...))
}

Ensure the handler has exactly one parameter of type RequestContext<T : RedisRequest>.
