Reports @HandleRedisRequest handler methods whose parameter list does not match the required signature.

A @HandleRedisRequest handler must declare exactly one parameter of type RequestContext<T>, where T is a subtype of RedisRequest. The RequestContext provides access to the incoming request object and the respond(...) method used to send a reply.

Examples of invalid signatures:


@HandleRedisRequest
fun handle() { }                                    // ERROR: no parameters

@HandleRedisRequest
fun handle(request: GetUserRequest) { }             // ERROR: not RequestContext

@HandleRedisRequest
fun handle(ctx: RequestContext<GetUserRequest>,
           extra: String) { }                       // ERROR: too many parameters

Correct signature:


@HandleRedisRequest
fun handle(context: RequestContext<GetUserRequest>) {
    context.respond(GetUserResponse("John"))
}

Fix the handler to accept exactly one RequestContext<T : RedisRequest> parameter.
