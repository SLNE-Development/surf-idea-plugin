Reports @OnRedisEvent handler methods whose parameter list does not match the required signature.

An @OnRedisEvent handler must declare exactly one parameter, and that parameter must be a subtype of RedisEvent. The framework uses this parameter to determine which event type the handler listens for and to pass the deserialized event object at runtime.

Examples of invalid signatures:


@OnRedisEvent
fun onEvent() { }                            // ERROR: no parameters

@OnRedisEvent
fun onEvent(event: RedisEvent, x: Int) { }   // ERROR: too many parameters

@OnRedisEvent
fun onEvent(message: String) { }             // ERROR: not a RedisEvent subtype

Correct signature:


@OnRedisEvent
fun onEvent(event: UserCreatedEvent) {
    println("User created")
}

Fix the handler to accept exactly one parameter of a RedisEvent subtype.
