Reports @HandleRedisRequest handler methods where the RequestContext parameter is not named context.

By convention, the RequestContext parameter in a request handler should be named context. Consistent naming improves readability and makes it easier to recognise handler patterns across the codebase.

Example:


// WARNING: parameter should be named 'context'
@HandleRedisRequest
fun handle(ctx: RequestContext<GetUserRequest>) {
    ctx.respond(GetUserResponse("John"))
}


// Correct
@HandleRedisRequest
fun handle(context: RequestContext<GetUserRequest>) {
    context.respond(GetUserResponse("John"))
}

The quick-fix renames the parameter to context.
